Final-year research project — AI-powered passenger safety & travel operations platform

Project Overview

The Travel Agent Supporting Tool is a comprehensive, AI-driven web application built to enhance passenger safety, monitor driver behaviour, detect alcohol impairment, and optimise route profitability for Sri Lankan travel agencies.

At its core, the system captures passenger images on entry and exit, then uses a cascade of deep-learning models — GAN-based reconstruction and a Siamese similarity network — to flag appearance anomalies in under five seconds, without disrupting the boarding flow.

Key Features

GAN-based Anomaly Detection

A Generator-Discriminator pair learns the distribution of normal passenger appearances. At exit, reconstruction error flags suspicious deviations, even for unseen anomaly types.

TensorFlow / Keras
Siamese Re-identification

Twin networks with shared weights compute cosine similarity between entrance and exit embeddings. A threshold of 0.7 determines whether the same passenger is confirmed.

128 x 128 RGB Input
Real-time Passenger Tracking

Flask-SocketIO pushes live events — passenger entry, exit, and anomaly alerts — directly to the dashboard without polling, keeping operators informed instantly.

WebSocket / Socket.IO
MongoDB Data Store

Five schema-validated collections — passengers, journeys, images, alerts, and system_logs — store every event with indexed queries for fast retrieval and auditing.

PyMongo Driver
Driver Ability Assessment

Monitors fatigue patterns and unsafe driving behaviours. Generates alerts and recommendations to maintain service quality and reduce accident risk.

ML Classification
Profit Assessment

Predicts route profitability using ML models trained on Sri Lankan bus operational data (40,000 records), surfacing actionable cost and revenue optimisation suggestions.

XGBoost / sklearn
